# 文明：星火纪元

六边形实时策略原型。玩家在开局前选择阵营、出生位置、地图和对战规模，进入战场后通过城市生产、工人开荒、科研解锁和单位指挥摧毁敌方核心城市。

在线页面：

https://chenxuan520.github.io/culture/

仓库：

https://github.com/chenxuan520/culture

## 运行

这是纯静态前端项目，可以直接打开 `index.html`，也可以用本地静态服务查看：

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000/
```

## 测试

```bash
npm test
```

测试覆盖资源/科技节奏、开局阵营、敌方 AI 行为、工人 AI、快捷键、地图生成、生产队列和基础交互。

## GitHub Pages

仓库使用 GitHub Actions 发布静态页面。推送到 `main` 后，`.github/workflows/pages.yml` 会上传整个仓库作为静态站点并部署到 GitHub Pages。

如果页面没有自动更新，检查 GitHub 仓库的 `Actions` 和 `Settings -> Pages`，Pages Source 应使用 `GitHub Actions`。

## 玩法核心

目标是发展文明并摧毁敌方首都。核心循环：

1. 城市自然获得少量基础资源。
2. 工人开荒，建设农场、矿山、伐木场等资源设施。
3. 科研解锁高级单位、建筑和设施。
4. 城市生产单位和建筑。
5. 编队推进，摧毁敌方城市与资源设施。

基础开荒不需要科技：农场、矿山、伐木场开局就能建。高级设施仍需要科技，例如星晶实验站、深层萃取井和日冕电站。

## 开局与资源

当前默认开局资源：

```text
食物 92
生产 72
科研 0
金币 118
能量 32
```

主城默认每次战略脉冲：

```text
食物 +1.5
生产 +1.4
科研 +1
金币 +3
能量 +0
```

主城基础增长很低，主要经济应来自工人开发地块和后续扩张。科研从 0 开始，第一批科技需要通过脉冲积累或开发星晶来推进。

## 阵营

玩家可以选择：

- 星火联盟
- 灰烬军团

选择某个阵营后，敌方自动使用另一阵营。玩家选择灰烬军团时，己方使用灰烬单位和红/黑/紫阵营色，敌方使用星火单位和蓝/绿/黄阵营色。

## 对战设置

首页的定制对局可以设置：

- 地图模式：默认地图或随机地图
- 玩家阵营
- 出生位置
- 敌军 AI 难度
- 盟友 AI 数量
- 敌方 AI 数量

左下角齿轮是局内设置：

- 音量
- 新工人默认 AI
- 开拓者到达后自动建城

`新工人默认 AI` 会影响开局工人和后续生产出的工人。

## 指挥规则

玩家手动命令优先级最高：

- 手动移动工人会清掉当前工地并关闭该工人的 AI。
- 手动移动士兵会暂停自动索敌，不会马上被自动追击抢走。
- 手动攻击敌人或设施会进入手动锁敌。
- 右键取消当前选择。


## AI 原则

电脑难度不靠开局资源、科技、血量、伤害或护甲领先。难度主要来自：

- 决策频率
- 扩张积极度
- 波次组织
- 目标选择
- 反制速度

中等和困难 AI 的开局资源与玩家一致，不自带开局科技、精英单位或首都护盾。
